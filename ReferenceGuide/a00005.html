Detailed Description

The ClientService API provides functionality for getting lists of clients or lists of client groups against which you can run the methods of other web services.

Client lists are returned as lists of computer GUIDs. Group lists are returned as lists of group IDs.

Clients can be specified by individual IP address or hostname, by lists of IP addresses or hostnames, or by the name of a Symantec Endpoint Protection Manager group (group name). For more information about groups, see the Symantec Endpoint Protection documentation.

The returned computer GUIDs are intended for use as input parameters to other web methods, especially the methods provided in the CommandService API and in the PolicyService API. Group IDs can be used as input parameters to the methods of the PolicyService API.

This API also includes methods to create a default PagingOptions object and a default Context object. These objects may be useful if you write your web service calls in PowerShell or another scripting language. Developers who use higher-level languages should be able to create these objects directly, or they can use these helper methods.

Returns a list of computer GUIDs from a provided list of computer hostnames.

The specified hostname can include the network domain name.

The computerHostNames parameter can be specified in any of the following formats:

  • hostname@domainName. Returns GUIDs for all computers that match both the hostname and the domain name
  • hostname. Returns GUIDs for all computers that match the hostname. Multiple GUIDs can be returned if the same hostname exists in multiple domains.
  • @domainName. If you specify the domain name, it must be preceded by the "@" symbol. Returns GUIDs for all computers in the specified domain.

GroupResult com.symantec.sepm.webservice.client.ClientService.getGroupsByName ( @WebParam(name="groupName") String  groupName,
@WebParam(name="pagingOptions") PagingOptions  pagingOptions,
@WebParam(name="context") Context  context 
) throws InsufficientAccessServiceException, InvalidArgumentServiceException, ServiceException

Returns a list of group IDs for Symantec Endpoint Protection Manager groups from the specified group name(s)

In addition to the group IDs, the returned GroupResult object provides the following:

  • A boolean object to indicate hasMoreResults
  • If hasMoreResults = true, then the context value to retrieve the next set of results

The groupName parameter can be specified in either of the following formats:

  • A complete group name path. Specifying the parameter in this format returns information only about the group whose name is specified. No information about child groups is returned. Example: MyCompany Group.
  • A pathname that includes a wildcard character. Only the asterisk (*) wildcard character is supported. It can be used only as the last character in the group name. Example: MyCompany*, MyCompany*
  • A wildcard character (*) used anywhere except as the last character causes an invalid argument exception.
  • A forward slash (/) or a backward slash () is accepted as a path delimiter.
