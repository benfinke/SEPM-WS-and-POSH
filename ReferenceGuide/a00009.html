Detailed Description

The CommandService API provides functionality for running commands on lists of Symantec Endpoint Protection client computers.

The methods of this API take a list of computer GUIDs that are generated by the methods of the ClientService API.

See also:
ClientService

Most of the methods of this API are used to run individual commands, such as Scan, Update, or enable/disable a protection technology. The getCommandStatusDetails method returns a CommandStatusDetailResult object that provides detailed information about the status of a specified command. This CommandStatusDetailResult object also contains a list of child objects that provide detailed information about the status of the command on each client in the list.

This API also includes methods to create a default PagingOptions object and a default Context object. These objects may be useful if you write your web service calls in PowerShell or another scripting language. Developers who use higher-level languages should be able to create these objects directly, or they can use these helper methods.

See also:
PagingOptions
Context

Note that the Custom scan type, which can be specified for the Scan command and for the Update Content and Scan command, is not defined in the API. It must be configured in the Symantec Endpoint Protection Manager console.

For more information about the commands that can be run on clients, see the Symantec Endpoint Protection documentation. For basic information about using commands or applying policies to remediate poor client health, see the Integration Guide.

CommandClientResult com.symantec.sepm.webservice.client.command.CommandService.runClientCommandDisableNTP ( @WebParam(name="computerGUIDList") String[]  computerGUIDList,
@WebParam(name="timelimit") long  timeLimit 
) throws InsufficientAccessServiceException, InvalidArgumentServiceException, ServiceException

Disables Network Threat Protection on a specified list of client computers.

The timeLimit parameter specifies the length of time after which Network Threat Protection is automatically reset to the state that is specified by the policy that is applied to the client. The minimum timeLimit that can be set is one minute, and the maximum is 720 minutes (12 hours). If a timeLimit greater than the maximum is specified, the maximum timeLimit is used.

Note that when the time limit is reached, Network Threat Protection is returned to the state that is specified by the client's policy. For example, the administrator might disable Network Threat Protection on a list of client computers some of which already have it disabled by policy. After the time limit is reached, the Network Threat Protection state that is specified by each client's policy is restored. Based on individual policies, some clients will re-enable Network Threat Protection, and some clients will continue in a disabled state.

Parameters:
computerGUIDListlist of computer GUIDs for client computers on which to disable Network Threat Protection

CommandStatusDetailResult com.symantec.sepm.webservice.client.command.CommandService.getCommandStatusDetails ( @WebParam(name="commandID") String  commandID,
@WebParam(name="pagingOptions") PagingOptions  pagingOptions,
@WebParam(name="context") Context  context 
) throws InsufficientAccessServiceException, InvalidArgumentServiceException, ServiceException

Gets detailed information about the status of a specified command for all clients on which the command was run.

If the pagingOptions parameter is null, this method uses the default values. The following values are the defaults:

  • DEFAULT_PAGE_SIZE (= 100 records)
  • SortOrder.ASCENDING
  • sortColumn.BeginTime

sortColumn can be one of the following:

  • beginTime
  • lastUpdateTime
  • stateId for the command
See also:
PagingOptions

If the context parameter is null, this method uses the default offset value, 1. Otherwise, this method uses the existing offset value from the context. The updated context object is returned to the caller by the CommandStatusDetailResult object. To retrieve the next set of results, use this updated context object.

See also:
Context

If there is no information to return in the CommandStatusDetailResult object, the fields that the object contains are returned as null values.
