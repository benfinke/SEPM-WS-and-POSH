Detailed Description

The Context class creates an object that that lets the client return its current state if multiple requests are required.

It is used together with a PagingOptions object to simplify retrieving large result sets.

For example, the lists of computer GUIDs or group names that are returned by the methods of the ClientService can be very large. If you have a result set of 10,000 records, and the page size is 1000, you need to make 10 calls to the same method to retrieve all the results.

A Context object can be used to provide the index (offset value) for the next method call.

Example:

 // First call to web service method
String groupName = "My Company";
 Options pagingOptions = new Options();
 Context context = new Context(1);
 GroupResult groupResult = ClientService.getGroupsByName(groupName, pagingOptions, context);
// Second call to web service method
 // Here we use the group result object that contains context and pagingOptions values provided by the initial call
groupResult = ClientService.getGroupsByName(groupName, pagingOptions, groupResult.getContext());

The same kind of call can be made to either of the ComputerResult methods (getComputersByIP, getComputersByHostName).

If the context parameter for any web method is null, an initial context object is created with the default offset (index) value.

If the context parameter is not null, the method uses the existing offset value from the context. The updated context object is returned to the caller through the parent object (GroupResult, ComputerResult). On return, the context object provides the offset value for updating the next call to the same method.
