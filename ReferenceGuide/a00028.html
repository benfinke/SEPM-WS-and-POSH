Validates the PagingOptions and Context objects for minimum and maximum page size, sort column, sort order, and offset minimum value.

Validation uses JSR-303 and explicit checks.

If a given PagingOptions is null, this method uses the default pageSize, sort order, and given sort column.

If a given context Context is null, this method uses the default offset value.

This method assumes a fixed sort column.

If validation succeeds, this method populates the paging options transfer object with pagination, sort, and offset values.

Parameters:
pagingOptionsobject that contains pagination information: page size, sort column, and sort order
contextobject that contains offset information
sortByColumnNamestring to indicate the column name on which to sort
sortByColumnNameValuestring in SQL database to indicate the column name on which to sort
validSortByColumnNamescolumn name values on which to sort
maxPageSizethe maximum page size. The maximum size that can be specified is 1000.
Returns:
PagingOptionsTO object that provides input values passed from the web service web method
Exceptions:
InvalidArgumentServiceExceptionif any pagination option is invalid

Validates the PagingOptions and Context objects for minimum and maximum page size, sort column, sort order, and offset minimum value, for the case where there are multiple possible values for the column on which to sort.

If validation succeeds, this method populates the paging options transfer object with pagination, sort, and offset values.
