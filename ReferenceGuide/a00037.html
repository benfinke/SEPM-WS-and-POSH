Detailed Description

The LiveUpdateService API provides functionality to manage LiveUpdate and to check the status of Symantec Endpoint Protection content (virus definitions) using an HTTPS SOAP request.

This service can be used to run LiveUpdate on a management server. It can also be used to check the date and the revision of the content that is available on the management server and on the Symantec LiveUpdate servers.

For more information about LiveUpdate functionality, see the Integration guide and the Symantec Endpoint Protection documentation.

LiveUpdateContentVersionResult com.symantec.sepm.webservice.liveupdate.LiveUpdateService.getLiveUpdateContentVersion ( ) throws InsufficientAccessServiceException, ServiceException

Gets information about LiveUpdate content.

This information consists of the following items:

  • The date and the revision of the virus definitions that are available on a specified management server. Provides an explicit error in the case that definitions are missing on the management server.
  • The date and the revision of the virus definitions that are available on Symantec LiveUpdate servers. Provides an explicit error in the case that definitions are unavailable from the Symantec LiveUpate servers.

int com.symantec.sepm.webservice.liveupdate.LiveUpdateService.runLiveUpdate ( ) throws InsufficientAccessServiceException, ServiceException

Runs LiveUpdate on the specified management server.

In the case where return value = 0 (LiveUpdate fails to start), the most common reason for a LiveUpdate failure is that LiveUpdate is already running on the management server.

Other less common reasons for failure are that management site replication is in progress, or that a client package import is in process. For more information about these processes, see the Symantec Endpoint Protection documentation.

Returns:
status of the service

0 = LiveUpdate session could not be started

1 = LiveUpdate session started successfully

Note that this status does not guarantee that the session will be successfully completed.

Exceptions:
InsufficientAccessServiceExceptionif the current logged in user does not have sufficient rights to access the endpoint or to execute the web method.

Only a system administrator or a domain administrator or limited administrator with full local site rights can run LiveUpdate.

Exceptions:
ServiceExceptionif the web service encountered an error processing the web request

error code 1 = the service cannot connect to the Endpoint Protection database

error code 2 = the license state is out of date
