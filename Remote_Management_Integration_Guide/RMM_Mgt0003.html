Remote management and monitoring of Symantec Endpoint Protection

This release of Symantec Endpoint Protection provides support for remote management and remote monitoring. Remote management is provided by means of public APIs in the form of web services that let you integrate your third-party solution or custom console with basic client and management server functionality. Remote monitoring is provided by means of publicly supported registry keys and Windows event logging.

License state and content state on the Symantec Endpoint Protection Manager server can be monitored by using either the Windows Event Log or the web services. Web services also provide client remediation.

See What's new in this release.

The new web services use the SOAP protocol over HTTPS. Secure communication is provided by means of the OAuth 2.0 protocol, which requires every web service call to include an access token that is tied to an individual administrator account on Symantec Endpoint Protection Manager. See Using OAuth with Symantec Endpoint Protection Manager web services.

Note:

These new web services are built on a new framework and are completely separate from earlier web services that were built for integration with Symantec Protection Center. If you already run tools that utilize these earlier web services, you can continue to run the tools. New tools should take advantage of the new web services.

You can also monitor services on the client and the server. We recommend that you monitor the services list directly, but services events are also written to the Windows Event Log.
