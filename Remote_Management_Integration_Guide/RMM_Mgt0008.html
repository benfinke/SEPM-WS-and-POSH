Creating a web services client application

This topic provides a high-level view of the steps required to set up your environment and create your web services client. For detailed information, see the linked topics.

Important: Access to Symantec Endpoint Protection Manager web services requires an individual administrator account on Symantec Endpoint Protection Manager. We do not recommend that you permit shared account access. There are three types of Symantec Endpoint Protection Manager administrator, with different levels of access rights. Calls to web services will fail with an InsufficientAcccessServiceException error if the administrator account does not have access rights to perform the action on the management server. Your application should handle this exception gracefully.

For example, only a system administrator has access to licensing information, in Symantec Endpoint Protection Manager or in a web service client. Also, a Symantec Endpoint Protection Manager administrator account of any type is always logged in to one Symantec Endpoint Protection Manager domain, and can manage only one domain at a time. The web service client can therefore retrieve information only about the current logged-in domain. For more information about the types of Symantec Endpoint Protection Manager administrator, see the product documentation.
