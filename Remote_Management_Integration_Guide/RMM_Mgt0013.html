Registering an application with Symantec Endpoint Protection Manager

To work with Symantec Endpoint Protection Manager web services, a web services client application must first be registered with the instance of Symantec Endpoint Protection Manager that the web services client will manage. Registration is performed on the Symantec Endpoint Protection Manager server, and a Symantec Endpoint Protection Manager administrator account of any type is required. Registration is performed only once for each instance of Symantec Endpoint Protection Manager.

In the case of a third-party web services client, this step is performed by a remote management administrator. The client application should provide a mechanism such as a form or web page for the administator to provide the required information to the application.

To register an application with Symantec Endpoint Protection Manager

  1. The following information is provided to the web services client application:

    • The IP address and web services port of the installed instance of Symantec Endpoint Protection Manager to be managed by the web services client. The default port number is 8446.

  2. The administrator logs on to the instance of Symantec Endpoint Protection Manager at the following URL:

    https://<hostname>:<port_number>/sepm/viewLoginRMM.do

    where <hostname> and <port_number> are the IP address or hostname and web services port of the installed instance of Symantec Endpoint Protection Manager. The default URL is the following:

    https://localhost:8446/sepm/viewLoginRMM.do

  3. The administrator registers the application on this page.
  4. The administrator provides the following information to the web services client:

    • The client ID and client secret that are returned when the application is registered.
