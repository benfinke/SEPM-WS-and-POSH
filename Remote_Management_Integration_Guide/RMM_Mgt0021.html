Static WSDL files

The static WSDL files and their paired XSDs are provided in the SDK folder, in the following location:

..\SEPM_WebService_SDK\WSDL\

Part of a sample WSDL file looks like the following:

<service name="ClientService">    
    <port name="ClientServicePort" binding="tns:ClientServicePortBinding">      
        <soap:address location="https://<hostname>:port_number/sepm/ws/v1/ClientService"/>;    
    </port>  
</service>

where hostname and port_number are the hostname or IP address and port number of the Symantec Endpoint Protection Manager with which a web service client application communicates.

Before your application can use these files, they must be modified for the environment where the application will run. If you are writing a third-party application, you must provide a mechanism for modifying them with your customer's values at runtime. You must also modify them for your own testing.

To modify the static WSDL files

  • In each WSDL file, replace the following string:

    localhost:8446

    with the hostname or IP address and web services port of Symantec Endpoint Protection Manager that your application communicates with.

    For example, if the hostname of a management server is MySEPMServer, and the port number is the default value, the WSDL element would look like the following:

    <soap:address location="https://MySEPMServer:8446 ...>

See Managing WSDL files for your web service calls.
