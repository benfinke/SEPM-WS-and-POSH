Dynamic WSDL files

The WSDL files can be generated dynamically from Symantec Endpoint Protection Manager. This approach automatically populates the hostname and port number with the appropriate values.

To generate a dynamic WSDL

  1. A Symantec Endpoint Protection Manager administrator logs on to the management console by using the following URL:

    https://<hostname>:<port_number>/sepm/

    where hostname is the hostname or IP address of the management server, and port_number is the port that was specified for web services access when the management server was installed. The default URL is:

    https://localhost:8446/sepm/

  2. If a security certificate warning appears, and if the URL is correct, choose the option to continue to the website.
  3. If necessary, the administrator logs on to Symantec Endpoint Protection Manager.

    Note:

    Any access to the Symantec Endpoint Protection Manager web services requires authentication. If the administrator does not have a current valid session on Symantec Endpoint Protection Manager, this step is required.
  4. The administrator enters URLs like the following examples. These URLs provide access to the WSDL and to the XSD for the specified web service.

    https://<hostname>:<port_number>/sepm/ws/v1/ClientService?wsdl

    https://<hostname>:<port_number>/sepm/ws/v1/ClientService?xsd=1

    Registration is not required for this step.

See Managing WSDL files for your web service calls.
