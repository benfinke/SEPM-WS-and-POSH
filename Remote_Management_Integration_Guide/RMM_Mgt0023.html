Constructing a web services client application from the WSDL files

You can create a web services client application in any language you choose. Sample PowerShell scripts are included in the SDK to help you get started, to test your application, and to help troubleshoot any issues. The scripts include calls to many of the web service methods, and a set of calls to fetch the access token for each administrator. They can be found in the following location:

..\SEPM_WebService_SDK\SampleCode\PowerShell\

Note:

These scripts are provided as a convenience only. See the legal disclaimer in the SDK.

See Using the sample PowerShell scripts.

The PowerShell scripts include samples to get an OAuth token and to include the token in calls to the web services.

See Using OAuth with Symantec Endpoint Protection Manager web services.

For detailed information about individual web methods, see the Reference Guide in the SDK.
