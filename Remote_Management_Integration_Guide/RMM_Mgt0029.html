Running commands on clients

If a threat outbreak occurs in a customer network, or if clients report an Infected or At Risk status, immediate remediation is best performed by running the appropriate commands on the affected clients.

To send a command to a client or group of clients, CommandService requires the following information:

  • A list of the client computers on which to run the specified command. This list is fetched from the ClientService. The list can be specified by the IP addresses or by the hostnames of individual clients.

  • The command method to run

    See also the Reference Guide in the SDK.

The command method returns a commandID that can be used to obtain more information about the client and the command that was run. For details, see the Reference Guide.
