Client commands reference

The following commands can be issued to the Symantec Endpoint Protection client to remediate issues that are detected while monitoring the network. See Monitoring client state and activity.

If you are a developer of a third-party remote management application, you may want to take some of this information into account as you design your user interface, or you may want to provide it to your customers.

Note:

Commands to disable individual technologies include a timeout parameter. After the timeout interval, the technology returns to the state that it was in before the command was issued. For more information, see the Reference Guide.

For more information about client commands and about the protection technologies that they manage, see the Symantec Endpoint Protection documentation.

Table: Client commands

Command

Definition

When and how to use

Scan

Scan the client computer, list of clients, or group of clients for threats or risks. You also specify the scan type: Active, Full, or Custom.

A custom scan must be defined in the Symantec Endpoint Protection Manager console.

If the client(s) report Infected but Up to date.

Update Content

Run LiveUpdate on the client(s).

If you want to update content on the client without running a scan at the same time

Update Content and Scan

Run LiveUpdate on the client(s) and then scan to check for any new risks.

If the client(s) report Infected and you are not certain that the client has the latest content.

Enable Auto-Protect

Enables Auto-Protect for the file system on the client(s)

Note:

Does not affect Auto-Protect for email.

If you let endpoint users disable Auto-Protect, or if you needed to disable it temporarily by policy from the management console.

There is no corresponding command to disable Auto-Protect because it should remain enabled most of the time. See About Symantec Endpoint Protection.

Enable Network Threat Protection and Disable Network Threat Protection

Enables or disables the firewall and intrusion prevention features on the client(s).

Network Threat Protection is designed to be compatible with most applications that use the network. On rare occasions, however, you may need to disable these technologies temporarily.

You enable Network Threat Protection if users have disabled it or after the requirements for which you disabled it are met.

Enable Download Insight and Disable Download Insight

Enables or disables download blocking based on file reputation

You may rarely need to disable Download Insight temporarily.

You enable Download Insight if users have disabled it or after the requirements for which you disabled it are met.

See Running commands on clients.
