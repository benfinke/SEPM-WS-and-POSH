Checking content from LiveUpdate on the server

By default, Symantec Endpoint Protection clients get content updates from Symantec Endpoint Protection Manager, which downloads content from the Symantec LiveUpdate servers.

The new LiveUpdateService API provides a method for checking on the status of content downloads from LiveUpdate to Symantec Endpoint Protection Manager. You can also monitor content status on the management server by using the Windows Event Log, as in previous versions.

If content on the server is reported as out-of-date, a new web method lets you run LiveUpdate on demand to get the latest content. See Running LiveUpdate on the server.

The LiveUpdateService web service reports on the status only of virus definitions. Other content types are also published, but virus definitions are published the most frequently - every four hours. They therefore provide a reliable indicator of overall content health.
