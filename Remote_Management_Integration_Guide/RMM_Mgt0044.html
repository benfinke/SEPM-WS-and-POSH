Monitoring license state on the management server

License monitoring is key to making sure that your endpoints are always protected. You can check Symantec Endpoint Protection licenses and Symantec Network Access Control licenses by monitoring a set of license events that are written to the Windows Event Log on the management server.

See Monitoring the management server.

See Server monitoring examples.

If you are not familiar with license management for Symantec Endpoint Protection, it may help to understand the product licensing model.

See About Symantec Endpoint Protection licensing.

Note:

There can be more than one valid purchased license. An individual license is associated with a specific number of seats. The license status becomes out of compliance (over-deployed) only if there are not enough total valid seats in all licenses.

License status

Summary of the current license status.

License status can be one of the following:

  • In compliance

  • About to expire

  • Over-deployed

  • Trial license is installed

  • Upgrade license is installed.

Use WMI to query for the most recent events from the Windows Event Log on the management server, Source Symantec Endpoint Protection, category Licensing, in the eventID range 7100 to 7105 for Symantec Endpoint Protection, and in the eventID range 7110 to 7115 for Symantec Network Access Control.

The most recent event in the range indicates the current license status. Each state is reported separately for Symantec Endpoint Protection and for Symantec Network Access Control.

See Licensing events.

License details

Information about all Symantec Endpoint Protection licenses.

Detailed information about each license is provided in the message text for the Licensing events. This information is useful to determine what kinds of licenses are installed, when they expire, and how to remediate in the case of overall over-deployed state or a license that is about to expire.

See Licensing events.
