About Symantec Endpoint Protection licensing

There are three types of Symantec Endpoint Protection licenses: trial, upgrade, and purchased.

Trial and upgrade licenses both contain an unlimited number of seats, but they both expire after specified periods of time.

A purchased license is associated with a specific number of seats and expires on a specified date. Both these variables (seats and expiration date) are determined at the time of purchase.

The license state is maintained on the management server, not on the client. Licenses persist on the server until they are manually removed, regardless of whether they are expired.

License state is out of compliance only if the number of installed clients exceeds the total number of purchased seats, regardless of the number of licenses with which those seats are associated.

Because of this model, it is possible to read multiple apparently conflicting licensing events on the server, even though the license state is ultimately in compliance. It is therefore very important to read the timestamp of a licensing event before you read the eventID. You may see an expired trial license, for example, in the case where there are also sufficient purchased licenses for the license state to be in compliance. The latter state is written later, so only its eventID is relevant to the curernt license state.
