Client events

The following events are logged on the client and supported in this release of Symantec Endpoint Protection. They are written to a custom target log for easier management (log retention, configuration, etc).

Note:

The first 17 events in this list (marked with an asterisk, *) are also logged in the Application Target Log, Event Source Symantec AntiVirus, for backward compatibility with earlier versions of the product. The only exceptions are events 13 and 14, which are logged to the application log only. We recommend that you monitor services directly. See Services running on the client and the server.

The content message text field contains the following subfields, in addition to the message text prefix that is specific to each eventID. Only the relevant subfields are written to the message text. For example, the source is written only to events that log download success or failure.

Table: Content message text details

Subfield label

Subfield description

Product

Symantec product (content type) registered with LiveUpdate.

Note:

This is the subfield that you must monitor to determine whether content is up to date on the server.

Version

The version number of the content type specified in the Product subfield.

Language

The language in which the specified content type is published.

Monikers

Server monikers, comma-separated

Sequence

Raw revision sequence

PublishDate

The date that this content type was published, in UTC complete date format.

Revision

[int]

Size (in bytes)

The size of the content package, in bytes (per content type)

Source

The source from which the client downloaded the specified content type (ENUM). Options are:

  • 1 = publicLU

    Public (Symantec) LiveUpdate server

  • 2 = privateLU

    LiveUpdate server installed on a customer's internal network. This type of LiveUpdate server is configured by using the LiveUpdate Administration utility.

  • 3 = localInstall

    Direct drop of content update (definitions) file (.vdb or .jdb) onto the client

  • 4 = SEPServer

    The management server, if it is configured to provide updated content to its clients

  • 5 = GUP

    A Group Update Provider (GUP) is a client that is designated to provide updates to other clients in a local network.
