Content revision state events on the management server

Content revision state events are written to the Windows Event Log on the management server and on the client.

Content revision state on the management server is important if the server is configured to provide client updates.

Note:

Content revision state events are written separately for each content type that the management server is registered to download. For a list of the available content types, see the Symantec Endpoint Protection documentation.

See Monitoring the management server.

The content message text field contains the following subfields, in addition to the message text prefix that is specific to each eventID. Only the relevant subfields are written to the message text. For example, the source is written only to events that log download success or failure.
